Vue_Mustache实现

发表于2021-08-20|更新于2021-08-20

Mustache译为胡子，因为{{ }}这样的语法糖老外说长得像胡子。

浅析

在Mustache中，一般会给render()传入一个模板字符串templateStr和数据data，render()将模板字符串里面的{{ }}转换为相应的数据，再用document.innerHTML进行赋值，就可以显示到页面上了。
​

总的而言，Mustache机制分为两个重要步骤：

  • 将templateStr解析为tokens；
  • 将tokens进行解析赋值，再将解析好的字符串返回。

index.html中的render()

将templateStr解析为tokens

总体思想：

  • 遍历templateStr；
  • 遇到{{`则把前面的字符串以`["text", String]`的形式存到`tokens`里面； - 遇到`}}则把{{ }}里面的数据以["name", String]的形式存到tokens里面；
  • 这样我们就得到了最初的tokens。

将tokens处理为nestedTokens

经过上一个步骤得到的tokens是最初始的tokens，我们需要处理Mustache语法中的#，将tokens变为nestedTokens。

总体思想：

  • 运用好collector变量，巧妙地使collector周转于stack的各个层级之中。

将tokens解析赋值

经过上一个步骤，我们得到了已经折叠好的tokens，是时候给它赋值了。

总体思想：

  • 遍历tokens，根据tag的不同分为以下几种情况：
    • tag == "text"：直接把token.value追加到htmlStr的结尾；
    • tag == "name"：如果{{ }}内为.，则直接将数据赋值给token.value；如果不是，则将data.value赋值给token.value；
    • tag == "#"：调用parseArray.js，在parseArray()中递归调用renderTemplate()
  • 返回htmlStr。

渲染页面

经过上一个步骤，我们得到了innerHTML，调用document.innerHTML，将HTML元素添加到指定id的container中，页面中就显示出来了。
